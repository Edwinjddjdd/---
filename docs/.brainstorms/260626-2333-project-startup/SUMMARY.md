# Brainstorm: 项目启动与阶段二落地

> Created: 2026-06-26 23:33:49

## Context

- 当前仓库已经包含阶段一研究资料，包括资料地图、建模依据、数据来源、课程笔记、深度阅读和论文笔记。
- 项目主题是常规单主旋翼带尾桨直升机的涡环状态建模与仿真。
- 当前缺口不是资料数量，而是缺少一个项目级入口，把研究资料转换成阶段二可执行的建模规格、验证路线和后续实现边界。
- 项目级约束要求所有 Markdown 中的数学表达式统一使用 LaTeX，例如状态方程应写成：

$$
\dot{x}=f(x,u,t)
$$

## Goals

- 建立项目总览，让后续工作能从一个稳定入口理解目标、范围、资料状态和关键约束。
- 将阶段一研究结论整理成阶段二建模规格草案，包括状态量、模块边界、VRS 判据接口和验证场景。
- 明确哪些内容已经可以进入规格，哪些仍需课程 PDF 或目标机型参数核验。
- 保持本轮改动只写文档和执行记录，不引入未经核验的仿真代码。

## Non-Goals

- 不实现 Simulink、MATLAB 或 Python 仿真模型。
- 不冻结中俄轴系的最终方向定义；该项仍依赖课程资料第 1、4 章原图核验。
- 不补造目标机型参数、翼型气动表或真实 VRS 试飞数据。
- 不改写既有研究笔记，只新增项目入口和阶段二草案。

## Chosen Approach

- 推荐采用“项目入口 + 阶段二规格草案 + 验证路线图”的最小落地方案。
- 理由：当前资料已经足够支撑模型结构和验证场景，但还不足以安全开始数值仿真代码。先把建模边界、依赖、接口和验收标准写清楚，可以减少后续实现时在轴系、符号和 VRS 判据上的返工。

## Alternatives Considered

- 直接开始写仿真代码：速度快，但中俄轴系、目标机型参数和验证数据尚未冻结，容易把错误假设固化进模型。
- 继续扩展文献阅读：能提高理论依据，但短期不会解决项目入口和阶段二执行边界缺失的问题。
- 只创建 `README.md`：入口更轻，但不足以承载状态向量、模块接口、VRS 判据和验证矩阵。
- 一次性写完整模型规格：完整度高，但会迫使当前未核验的轴系和参数被过早定稿。

## Risks & Mitigations

- 风险：阶段二草案被误用为最终数学规格。缓解：在文档中明确标注“草案”和“待冻结项”。
- 风险：VRS 判据过度依赖 FAA 定性条件。缓解：将 FAA 判据定位为初始触发逻辑，将 NASA/Johnson 模型列为后续修正依据。
- 风险：没有 `docs/SUMMARY.md` 导致后续代理重复扫全仓库。缓解：本轮新增项目总览并链接核心研究文件。
- 风险：数学表达式格式不统一。缓解：所有新增文档中变量、矩阵、方程和上下标均使用 LaTeX。

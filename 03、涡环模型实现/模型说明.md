# 涡环入流模型说明

## 1. 当前模型做了什么

当前实现是一个独立的涡环平均诱导速度测试台，不包含六自由度机体。它完成：

$$
\begin{aligned}
\left(V_H,V_d,T\right)
&\longrightarrow \left(v_h,\mu,\nu\right)\\
&\longrightarrow \left(\bar v_{\mathrm{base}},x_g,g_{\mathrm{VRS}}\right)\\
&\longrightarrow \bar q_{\mathrm{target}}\text{ 的 D-N-X-E 分段曲线}\\
&\longrightarrow \Delta v_{\mathrm{VRS}}\\
&\longrightarrow v_{i,\mathrm{QS}}\\
&\xrightarrow{\tau\text{ 与积分}}v_i\\
&\longrightarrow q=v_i-V_d.
\end{aligned}
$$

对应的物理过程是：根据当前速度和拉力计算基线诱导速度，加入 Johnson VRS 修正得到准定常目标，再经过尾流动态滞后得到实际诱导速度，最后与下降造成的相对来流合成总轴向入流。

其中水平速度门控量为：

$$
x_g=\operatorname{clip}\left(
\frac{\mu}{\max(\mu_M,\varepsilon_{\mathrm{tol}})},0,1
\right),
$$

$$
g_{\mathrm{VRS}}=1-x_g^2(3-2x_g).
$$

$x_g$ 是“关断进度”，$g_{\mathrm{VRS}}$ 才是保留下来的 VRS 修正比例。
当 $mu=0$ 时 $g_{\mathrm{VRS}}=1$；当 $mu\ge\mu_M$ 时
$g_{\mathrm{VRS}}=0$。代码中的 `tol` 只防止除零，不代表新的物理边界。

Simulink 模型为 [`vrs_inflow_testbench.slx`](./vrs_inflow_testbench.slx)。

### 1.1 为什么会有这些程序

本项目不是“每个文件各算一遍”，而是把参数、准定常空气动力、动态响应和测试分开。完整调用关系是：

~~~text
vrs_params.m
  产生统一参数 P 和 Simulink 数值向量 P.vector
        |
        v
vrs_johnson_qs.m                 <- 给人读、给 MATLAB 测试脚本调用
        |
        v
vrs_johnson_qs_numeric.m         <- 唯一的准定常 VRS 数值核心
        |
        v
v_i_QS                           <- 当前工况保持不变时的目标诱导速度
        |
        v
vrs_inflow_rhs.m / Simulink 动态块
        |
        v
Integrator                       <- 唯一真正保存动态状态的地方
        |
        v
v_i                              <- 当前实际诱导速度
~~~

外围脚本不重复定义模型：

~~~text
init_vrs_model.m
  只负责准备 P_vrs、T_ref、vi0

test_vrs_static.m
  只负责验证准定常曲线、门控、连接点和开关

run_vrs_testbench.m
  只负责生成时间输入、运行 slx、提取结果和画图

plot_vrs_6dof_comparison.m
  只负责把未来六自由度模型的结果画成对比图

open_vrs_model.m
  只负责初始化并打开模型
~~~

因此，看到一个程序时先问它属于哪一层：

| 层次 | 回答的问题 | 对应文件 |
|---|---|---|
| 参数层 | 模型用什么旋翼、边界和时间常数？ | vrs_params.m |
| 静态空气动力层 | 当前工况若保持不变，目标诱导速度是多少？ | vrs_johnson_qs_numeric.m |
| 易读接口层 | 怎样把数值核心的多个输出整理成人能读的字段？ | vrs_johnson_qs.m |
| 动态层 | 实际尾流以多快速度追上目标？ | vrs_inflow_rhs.m、Simulink Integrator |
| 初始化层 | 模型运行前工作区必须有哪些量？ | init_vrs_model.m |
| 验证层 | 数学曲线是否连续、开关是否有效？ | test_vrs_static.m |
| 场景层 | 怎样制造进入与改出 VRS 的时间过程？ | run_vrs_testbench.m |

### 1.2 推荐阅读顺序

第一次阅读不要从 run_vrs_testbench.m 的绘图部分倒推算法，按下面顺序最顺：

1. 读 vrs_params.m，只认识输入参数和 D、N、X、E、$\mu_C$、$\mu_M$。
2. 读 vrs_johnson_qs.m，先看这个模型总体输出哪些物理量。
3. 读 vrs_johnson_qs_numeric.m 的第 1～7 步，这是核心建模流程。
4. 再读同一文件末尾的辅助函数，理解每一步如何实现。
5. 读 vrs_inflow_rhs.m，区分准定常目标 $v_{i,\mathrm{QS}}$ 和动态状态 $v_i$。
6. 读 test_vrs_static.m，看每条公式如何被验证。
7. 最后读 run_vrs_testbench.m 和 Simulink 模型，看时间场景和动态积分。

### 1.3 核心程序为何按这个顺序计算

vrs_johnson_qs_numeric.m 内部的 7 步不是编程习惯，而是物理依赖关系决定的：

1. 先检查参数，因为无效的 $T$、$R$、$\rho$ 或 $\Omega$ 会使后续公式失去意义。
2. 再计算 $v_h$，因为 $\mu$、$\nu$、D、N、X、E 全部建立在无量纲速度上。
3. 再计算基线，因为 VRS 修正描述的是“相对正常尾流状态偏离多少”。
4. 再计算水平门控，因为水平速度决定 VRS 修正应保留多少、N/X 区域应收缩多少。
5. 再构造 D-N-X-E 曲线，因为只有知道当前 $\mu$ 和 $\nu$ 后才能选择分段。
6. 再从总入流曲线反算诱导速度，因为 Johnson 控制点给的是总入流 $q$，模型接口需要的是诱导速度 $v_i$。
7. 最后才形成 $v_{i,\mathrm{QS}}$；它仍只是目标值，必须交给动态方程积分得到实际 $v_i$。

这条依赖链可写为：

$$
T
\longrightarrow v_h
\longrightarrow (\mu,\nu)
\longrightarrow \bar v_{\mathrm{base}}
\longrightarrow (x_g,g_{\mathrm{VRS}},\nu_N^*,\nu_X^*)
\longrightarrow \bar q_{\mathrm{target}}
\longrightarrow \Delta v_{\mathrm{VRS}}
\longrightarrow v_{i,\mathrm{QS}}
\longrightarrow \dot v_i
\longrightarrow v_i.
$$

如果程序中某一行无法解释它服务于这条链中的哪一个箭头，就不应该直接保留，而应补充物理说明或拆成命名清楚的辅助函数。

## 2. 每个方块的物理意义

| 方块 | 物理问题 |
|---|---|
| `Johnson_VRS_QS` | 当前速度和拉力下，如果工况保持不变，诱导速度最终应该是多少？ |
| `Target_Minus_Actual` | 目标诱导速度与当前实际诱导速度还差多少？ |
| `Divide_By_Tau` | 真实尾流应该以多快的速度追赶目标？ |
| `Actual_Induced_Velocity` | 对变化率积分，得到当前实际诱导速度 $v_i$ |
| `Total_Axial_Inflow` | 把诱导速度与下降造成的相对来流合成，得到 $q=v_i-V_d$ |

## 3. 输入与输出

输入：

| 信号 | 含义 | 单位 | 正方向 |
|---|---|---|---|
| `V_H_mps` | 桨盘面内速度 | $\mathrm{m/s}$ | 大小为正 |
| `V_d_mps` | 下降率 | $\mathrm{m/s}$ | 向下为正 |
| `T_N` | 主旋翼拉力 | $\mathrm{N}$ | 正值 |

输出：

| 信号 | 含义 | 单位 |
|---|---|---|
| `v_i_mps` | 实际动态诱导速度 $v_i$ | $\mathrm{m/s}$ |
| `v_i_QS_mps` | 准定常目标诱导速度 $v_{i,\mathrm{QS}}$ | $\mathrm{m/s}$ |
| `vrs_flag` | 当前是否位于 VRS 修正区 | `boolean` |
| `q_mps` | 桨盘总轴向入流 $q=v_i-V_d$ | $\mathrm{m/s}$ |
| `v_base_mps` | 基线诱导速度 $v_{\mathrm{base}}$ | $\mathrm{m/s}$ |
| `dv_vrs_mps` | Johnson VRS 诱导速度修正量 | $\mathrm{m/s}$ |

## 4. 文件作用

| 文件 | 作用 |
|---|---|
| `vrs_params.m` | 集中保存旋翼、环境、Johnson 边界和时间常数参数 |
| `vrs_johnson_qs_numeric.m` | 可供 MATLAB Function 块调用的数值算法 |
| `vrs_johnson_qs.m` | 便于人工阅读和 MATLAB 扫掠的结构体接口 |
| `vrs_inflow_rhs.m` | 一阶动态入流方程 |
| `init_vrs_model.m` | 初始化参数、参考拉力和积分器初值 |
| `test_vrs_static.m` | 静态入流曲线、VRS 边界、连接连续性和 N/X 零斜率检查 |
| `run_vrs_testbench.m` | 使用现有模型运行进入与水平改出场景并绘图 |
| `plot_vrs_6dof_comparison.m` | 接收六自由度仿真数据并绘制综合对比图 |
| `open_vrs_model.m` | 初始化并打开顶层模型和 `VRS_Inflow` 子系统 |

## 5. 怎么运行

在 MATLAB 中进入本目录，依次运行：

```matlab
test_vrs_static
run_vrs_testbench
```

上述脚本直接使用已有的 `vrs_inflow_testbench.slx`，不会重新生成模型。

只想打开模型时运行：

```matlab
open_vrs_model
```

## 6. 当前演示场景

| 时间 | 工况 | 物理意义 |
|---|---|---|
| $0\sim5\ \mathrm{s}$ | $V_H=0$、$V_d=0$ | 悬停 |
| $5\sim15\ \mathrm{s}$ | $V_d$ 增加到 $11\ \mathrm{m/s}$ | 垂直下降进入 VRS |
| $15\sim25\ \mathrm{s}$ | 保持下降率 | 观察 VRS 动态建立 |
| $25\sim35\ \mathrm{s}$ | $V_H$ 增加到 $15\ \mathrm{m/s}$ | 水平速度把尾流带离桨盘 |
| $35\sim40\ \mathrm{s}$ | 保持水平速度 | 检查 VRS 标志退出 |

## 9. 参数边界

当前使用 Johnson 报告 D6075 算例参数：旋翼半径 $R=5.97\ \mathrm{m}$、转速 $360\ \mathrm{r/min}$、演示质量 $3500\ \mathrm{kg}$。这些数据仅用于跑通模型结构和复现典型趋势，不代表最终目标直升机。

换目标机时首先修改 `vrs_params.m`，不修改 Simulink 连线。Johnson 基线水平速度混合和 D-N-X-E Hermite 分段属于第一版工程实现，最终报告使用前仍需根据原报告曲线和目标旋翼数据继续校核。
